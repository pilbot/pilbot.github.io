Why Revmoji

We don’t need to argue why code reviews are so critical to a software project. Many intelligent people have answered that question comprehensively. Instead I’d like to focus on code review as a tool for encouraging shared ownership and training people on a code base.

Typically we’ll work on teams with a broad range of abilities, this often leads to an imbalance when reviewing. More junior engineers can be apprehensive about reviewing a more senior (or project lead’s) code. That is if they even get the opportunity of review at all.

Conversely when code is reviewed it can be difficult for the proposer to tease out what is a stylistic point and what is a critical concern. With an experience or skill disparity this is exacerbated, resulting in longer times to merge.

Using emoji we can quickly and easily mark our code reviews in a way that both parties can understand. Hopefully this system just acts as a trigger for developing a much broader shared language with your team.
